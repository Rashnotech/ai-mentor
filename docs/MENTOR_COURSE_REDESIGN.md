# Mentor Course Section - Comprehensive Redesign

> **Version:** 2.0  
> **Date:** February 3, 2026  
> **Author:** Senior Product Engineer

---

## Overview

This document defines the complete data models, entity relationships, admin UI behavior, and business rules for the Mentor Course Management section. The design supports:

- One course â†’ Multiple learning paths
- One default learning path per course
- Modules shared across multiple learning paths
- Lessons, Quizzes, and Projects linked to modules
- Mentor submission tracking for projects

---

## 1. Data Model Definitions

### 1.1 Course Model

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `course_id` | Integer (PK) | Auto | Primary key |
| `title` | String(255) | Yes | Course title |
| `description` | Text | Yes | Course description |
| `slug` | String(100) | Yes | URL-friendly unique identifier |
| `cover_image_url` | String(255) | No | Cover image URL |
| `estimated_hours` | Integer | No | Total estimated hours |
| `difficulty_level` | Enum | No | BEGINNER, INTERMEDIATE, ADVANCED |
| `is_active` | Boolean | Yes | Default: true |
| `prerequisites` | String[] | No | List of prerequisite topics |
| `what_youll_learn` | String[] | No | Learning outcomes |
| `certificate_on_completion` | Boolean | No | Default: false |
| `average_rating` | Integer | No | Cached rating (0-50, where 45 = 4.5) |
| `total_reviews` | Integer | No | Review count |
| `created_by` | String(36) FK | No | User ID of creator |
| `created_at` | DateTime | Auto | Creation timestamp |
| `updated_at` | DateTime | Auto | Last update timestamp |

---

### 1.2 Learning Path Model

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `path_id` | Integer (PK) | Auto | Primary key |
| `course_id` | Integer (FK) | Yes | Reference to Course |
| `title` | String(255) | Yes | Path title |
| `description` | Text | No | Path description |
| `price` | Decimal(10,2) | No | Path price (default: 0.00) |
| `is_default` | Boolean | Yes | **Only ONE per course** (default: false) |
| `is_custom` | Boolean | No | Custom path flag (default: false) |
| `min_skill_level` | Enum | No | Minimum skill level |
| `max_skill_level` | Enum | No | Maximum skill level |
| `tags` | String[] | No | Categorization tags |
| `created_by` | String(36) FK | No | Creator user ID |
| `created_at` | DateTime | Auto | Creation timestamp |
| `updated_at` | DateTime | Auto | Last update timestamp |

**Business Rules:**
- `is_default = true` can only exist for ONE path per course
- When setting a new path as default, automatically unset the previous default
- Price must be >= 0

---

### 1.3 Module Model (NEW: Multi-Path Support)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `module_id` | Integer (PK) | Auto | Primary key |
| `title` | String(255) | Yes | Module title |
| `slug` | String(100) | Yes | URL-friendly unique identifier |
| `description` | Text | No | Module description |
| `order` | Integer | Yes | Display/sequence order |
| `estimated_hours` | Integer | No | Estimated completion hours |
| `unlock_after_days` | Integer | Yes | Days after enrollment to unlock (default: 0) |
| `is_available_by_default` | Boolean | Yes | Immediate availability (default: true) |
| `first_deadline_days` | Integer | No | Days to 100% score deadline |
| `second_deadline_days` | Integer | No | Days to 50% score deadline |
| `third_deadline_days` | Integer | No | Days to 25% score deadline |
| `auto_generate_slug` | Boolean | No | Auto-generate slug from title |
| `created_at` | DateTime | Auto | Creation timestamp |
| `updated_at` | DateTime | Auto | Last update timestamp |

**NEW: Module-Path Junction Table**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | Integer (PK) | Auto | Primary key |
| `module_id` | Integer (FK) | Yes | Reference to Module |
| `path_id` | Integer (FK) | Yes | Reference to LearningPath |
| `order_in_path` | Integer | Yes | Order within this specific path |
| `created_at` | DateTime | Auto | Creation timestamp |

**Business Rules:**
- `unlock_after_days` must be >= 0
- Deadline chronology: `first_deadline_days` < `second_deadline_days` < `third_deadline_days`
- Module slug must be unique across the system
- A module can belong to multiple learning paths with different ordering

---

### 1.4 Lesson Model

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `lesson_id` | Integer (PK) | Auto | Primary key |
| `module_id` | Integer (FK) | Yes | Reference to Module |
| `title` | String(255) | Yes | Lesson title |
| `description` | Text | No | Brief description |
| `content` | Text | No | Rich text/Markdown content |
| `content_type` | Enum | No | theory, coding, debugging, quiz |
| `order` | Integer | Yes | Order within module |
| `estimated_minutes` | Integer | No | Estimated completion time |
| `youtube_video_url` | String(500) | No | YouTube video URL |
| `external_resources` | String[] | No | Array of resource URLs |
| `expected_outcomes` | String[] | No | Learning outcomes |
| `starter_file_url` | String(255) | No | Starter code URL |
| `solution_file_url` | String(255) | No | Solution code URL |
| `created_at` | DateTime | Auto | Creation timestamp |
| `updated_at` | DateTime | Auto | Last update timestamp |

**Business Rules:**
- `order` must be unique per module
- `content_type` enum: `theory`, `coding`, `debugging`, `quiz`
- If `content_type` = `quiz`, lessons should link to quiz questions

---

### 1.5 Quiz Model (Assessment Questions)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `question_id` | Integer (PK) | Auto | Primary key |
| `module_id` | Integer (FK) | Yes | Reference to Module |
| `lesson_id` | Integer (FK) | No | **NEW:** Optional link to Lesson |
| `question_text` | Text | Yes | The question |
| `question_type` | Enum | Yes | single_choice, multiple_choice, true_false |
| `difficulty_level` | Enum | No | easy, medium, hard |
| `order` | Integer | Yes | Display order |
| `options` | String[] | Conditional | Answer options (required for choice types) |
| `correct_answers` | String[] | Yes | **CHANGED:** Array for multi-select support |
| `explanation` | Text | No | Explanation for correct answer |
| `points` | Integer | Yes | Points awarded (default: 10) |
| `created_at` | DateTime | Auto | Creation timestamp |
| `updated_at` | DateTime | Auto | Last update timestamp |

**Business Rules:**
- `points` must be > 0
- All values in `correct_answers` must exist in `options`
- For `single_choice` and `true_false`, `correct_answers` has exactly 1 element
- For `multiple_choice`, `correct_answers` can have multiple elements

---

### 1.6 Project Model

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `project_id` | Integer (PK) | Auto | Primary key |
| `module_id` | Integer (FK) | Yes | Reference to Module |
| `lesson_id` | Integer (FK) | No | **NEW:** Optional link to Lesson |
| `title` | String(255) | Yes | Project title |
| `description` | Text | Yes | Instructions/requirements |
| `order` | Integer | Yes | Display order |
| `estimated_hours` | Integer | No | Estimated completion hours |
| `starter_repo_url` | String(255) | No | Starter repository URL |
| `solution_repo_url` | String(255) | No | Solution repository URL |
| `required_skills` | String[] | No | Required skills list |
| `first_deadline_days` | Integer | No | Days to 100% score deadline |
| `second_deadline_days` | Integer | No | Days to 50% score deadline |
| `third_deadline_days` | Integer | No | Days to 25% score deadline |
| `created_at` | DateTime | Auto | Creation timestamp |
| `updated_at` | DateTime | Auto | Last update timestamp |

---

### 1.7 Project Submission Model (NEW)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `submission_id` | Integer (PK) | Auto | Primary key |
| `project_id` | Integer (FK) | Yes | Reference to Project |
| `student_id` | String(36) FK | Yes | Student user ID |
| `submission_url` | String(500) | No | Student's submission URL |
| `submitted_at` | DateTime | No | Student submission timestamp |
| `status` | Enum | Yes | pending, submitted, reviewed, approved, rejected |
| `mentor_id` | String(36) FK | No | Mentor who reviewed |
| `mentor_name` | String(255) | No | Mentor display name (cached) |
| `reviewed_at` | DateTime | No | Mentor review timestamp |
| `mentor_feedback` | Text | No | Mentor comments |
| `score` | Integer | No | Score assigned (0-100) |
| `score_multiplier` | Decimal | No | Deadline multiplier (1.0, 0.5, 0.25) |
| `created_at` | DateTime | Auto | Creation timestamp |
| `updated_at` | DateTime | Auto | Last update timestamp |

**Business Rules:**
- Mentor submission metadata required for project completion
- Score multiplier applied based on submission deadline
- Status flow: `pending` â†’ `submitted` â†’ `reviewed` â†’ `approved`/`rejected`

## 4. Key Validation and Business Rules

### 4.1 Learning Path Rules

| Rule | Description | Error Message |
|------|-------------|---------------|
| Unique Default | Only one path can be `is_default=true` per course | "Another path is already set as default" |
| Auto-Unset | When setting new default, previous is automatically unset | (Silent operation) |
| Price Validation | Price must be >= 0 | "Price cannot be negative" |
| Title Required | Title is mandatory | "Path title is required" |

### 4.2 Module Rules

| Rule | Description | Error Message |
|------|-------------|---------------|
| Path Assignment | At least one learning path must be selected | "Select at least one learning path" |
| Unique Slug | Slug must be unique system-wide | "This slug is already taken" |
| Slug Format | Only lowercase, numbers, hyphens allowed | "Invalid slug format" |
| Unlock Days | Must be >= 0 | "Unlock days cannot be negative" |
| Deadline Order | first < second < third deadline days | "Deadlines must be in chronological order" |
| Order Positive | Order must be > 0 | "Order must be a positive number" |

### 4.3 Lesson Rules

| Rule | Description | Error Message |
|------|-------------|---------------|
| Unique Order | Order must be unique within the module | "This order number is already used" |
| Title Required | Title is mandatory | "Lesson title is required" |
| Valid Content Type | Must be one of: theory, coding, debugging, quiz | "Invalid content type" |
| Valid YouTube URL | If provided, must be a valid YouTube URL | "Invalid YouTube URL format" |

### 4.4 Quiz Rules

| Rule | Description | Error Message |
|------|-------------|---------------|
| Positive Points | Points must be > 0 | "Points must be greater than 0" |
| Valid Answers | All correct_answers must exist in options | "Correct answer not found in options" |
| Single Choice | For single_choice, exactly 1 correct answer | "Single choice requires exactly one answer" |
| True/False Options | For true_false, options = ["True", "False"] | "True/False must have exactly two options" |
| Options Required | For choice types, at least 2 options required | "At least 2 options are required" |

### 4.5 Project Rules

| Rule | Description | Error Message |
|------|-------------|---------------|
| Title Required | Title is mandatory | "Project title is required" |
| Description Required | Instructions are mandatory | "Project description is required" |
| Valid URLs | Repository URLs must be valid if provided | "Invalid repository URL" |
| Mentor Required | All submissions must have mentor metadata | "Mentor review is required" |

## 7. Database Schema Changes Required

### 7.1 New Table: `module_path_assignments`

```sql
CREATE TABLE module_path_assignments (
    id SERIAL PRIMARY KEY,
    module_id INTEGER NOT NULL REFERENCES modules(module_id) ON DELETE CASCADE,
    path_id INTEGER NOT NULL REFERENCES learning_paths(path_id) ON DELETE CASCADE,
    order_in_path INTEGER NOT NULL DEFAULT 1,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(module_id, path_id)
);

CREATE INDEX idx_mpa_module ON module_path_assignments(module_id);
CREATE INDEX idx_mpa_path ON module_path_assignments(path_id);
```

### 7.2 New Table: `project_submissions`

```sql
CREATE TABLE project_submissions (
    submission_id SERIAL PRIMARY KEY,
    project_id INTEGER NOT NULL REFERENCES projects(project_id) ON DELETE CASCADE,
    student_id VARCHAR(36) NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    submission_url VARCHAR(500),
    submitted_at TIMESTAMPTZ,
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    mentor_id VARCHAR(36) REFERENCES users(id) ON DELETE SET NULL,
    mentor_name VARCHAR(255),
    reviewed_at TIMESTAMPTZ,
    mentor_feedback TEXT,
    score INTEGER,
    score_multiplier DECIMAL(3,2) DEFAULT 1.00,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_ps_project ON project_submissions(project_id);
CREATE INDEX idx_ps_student ON project_submissions(student_id);
CREATE INDEX idx_ps_mentor ON project_submissions(mentor_id);
CREATE INDEX idx_ps_status ON project_submissions(status);
```
